# サブステートマシン使用方法

サブステートマシンは、**ステート内にさらに細かいステートを持たせる**機能です。  
例えば「攻撃ステート」の中に「予備動作」「攻撃実行」「硬直」といった段階を作れます。

---

## 基本的な使い方

### 1. 親ステートでサブステートを追加

```csharp
public class S_Attack : State<AITester>
{
    public S_Attack(AITester owner) : base(owner) { }

    public override void Enter()
    {
        // サブステートを追加（インデックスは追加順）
        AddSubState(new S_Attack_Start(owner));   // インデックス 0
        AddSubState(new S_Attack_Exec(owner));    // インデックス 1
        AddSubState(new S_Attack_End(owner));     // インデックス 2

        // 最初のサブステートへ遷移
        ChangeSubState(0);
    }
}
```

### 2. サブステートクラスを作成

```csharp
public class S_Attack_Start : State<AITester>
{
    // ワールド変数（メンバー変数）
    private float m_Timer = 0f;
    private float m_Duration = 0.5f;

    public S_Attack_Start(AITester owner) : base(owner) { }

    public override void Enter()
    {
        m_Timer = 0f;
        Debug.Log("予備動作開始");
    }

    public override void Stay()
    {
        m_Timer += Time.deltaTime;
        
        if (m_Timer >= m_Duration)
        {
            // 親ステート経由で次のサブステートへ遷移
            GetParentState()?.ChangeSubState(1);
        }
    }

    public override void Exit()
    {
        Debug.Log("予備動作終了");
    }
}
```

---

## 処理の流れ

```
親ステート S_Attack の Enter() 実行
    ↓
AddSubState() でサブステート登録
    ↓
ChangeSubState(0) で S_Attack_Start の Enter() 実行
    ↓
毎フレーム: S_Attack.Stay() → S_Attack_Start.Stay()
    ↓
条件を満たしたら GetParentState()?.ChangeSubState(1)
    ↓
S_Attack_Start.Exit() → S_Attack_Exec.Enter()
    ↓
...繰り返し...
    ↓
最後のサブステートで owner.RevertToPreviousState() で親ステート終了
```

---

## 主要メソッド一覧

| メソッド | 説明 | 使用場所 |
|---------|------|---------|
| `AddSubState(state)` | サブステートを追加 | 親ステートの`Enter()` |
| `ChangeSubState(index)` | インデックスでサブステートに遷移 | 親・子どちらでも可 |
| `GetParentState()` | 親ステートを取得 | サブステート内 |
| `HasSubStates()` | サブステートを持っているか | どこでも |
| `GetCurrentSubState()` | 現在のサブステートを取得 | 親ステート |

---

## ポイント

1. **サブステートからの遷移**は `GetParentState()?.ChangeSubState(index)` を使う
2. **親ステートへ戻る**場合は `owner.RevertToPreviousState()` で全体を終了
3. サブステートの`Stay()`は、`StateMachine.Update()`内で**自動的に呼ばれる**（親の`Stay()`の後）
